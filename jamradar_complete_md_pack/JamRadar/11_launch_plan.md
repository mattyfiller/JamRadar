# 11 - Launch Plan

## Launch Strategy

Start in one region before expanding.

The app needs density. A weak national launch with scattered events is worse than a strong local launch with lots of relevant events.

## Best First Region Options

Since the founder is in Canada, possible test regions include:

- Ontario ski hills
- Banff/Canmore
- Whistler/Squamish/Vancouver
- Quebec resorts
- Interior BC

Ontario could be a practical starting region because many users are within driving distance of multiple smaller hills, indoor training options, shops, and skateparks.

## Pre-Launch Steps

1. Choose launch region.
2. Build list of mountains, indoor facilities, shops, clubs, and skateparks.
3. Manually collect public event information.
4. Build landing page.
5. Create Instagram/TikTok account.
6. Start posting weekly event roundups.
7. Contact local shops and mountains.
8. Recruit early users through communities.
9. Collect email waitlist.

## Launch Database Goal

Before launch, seed the app with:

- 50+ events
- 20+ venue profiles
- 10+ shops/facilities
- 10+ gear deals or gear-related posts

## Early Organizer Outreach

Target:

- Local ski hills
- Park managers
- Indoor ski/snowboard facilities
- Ski/snowboard shops
- Skate shops
- Event organizers
- Clubs
- Camps

Pitch:

“JamRadar helps riders near you find your events without relying only on Instagram algorithms.”

## Early User Acquisition

Methods:

- TikTok clips of local event roundups
- Instagram Reels
- Reddit/local groups
- Discord groups
- Posters at shops
- QR codes at resorts
- Partner shops
- Event sponsorships
- Campus ski/snowboard clubs

## Launch Offer for Organizers

Offer free posting for the first season.

Charge later for:

- Featured placement
- Advanced analytics
- Push promotion
- Deal promotion

## Launch Offer for Shops

Offer:

- Free profile
- Free first deal post
- Free event posting

Then upsell:

- Featured deals
- Sponsored local feed
- Gear swap promotion

## 90-Day Launch Plan

### Days 1-30

- Finalize MVP design
- Build landing page
- Build event database manually
- Contact organizers
- Collect users

### Days 31-60

- Launch beta
- Add 50-100 events
- Test notifications
- Get feedback
- Improve filters
- Add indoor training listings

### Days 61-90

- Add gear deals
- Start sponsored placements
- Expand event categories
- Improve organizer dashboard
- Prepare second region

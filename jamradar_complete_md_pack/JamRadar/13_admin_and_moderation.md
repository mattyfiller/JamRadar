# 13 - Admin and Moderation

## Why Admin Tools Matter

JamRadar depends on trust. If event listings are wrong, spammy, or outdated, users will stop relying on the app.

Admin tools should exist early, even if simple.

## Admin Responsibilities

Admins need to:

- Approve organizations
- Approve user-submitted events
- Edit incorrect event info
- Remove spam
- Verify mountains, shops, and facilities
- Handle reports
- Feature events
- Review gear deals
- Moderate marketplace listings

## Event Verification Levels

### Official

Posted by a verified organization.

### Claimed

Originally submitted by a user, then claimed by an organizer.

### Community Submitted

Submitted by a user and approved by admin.

### Unverified

Should not be shown broadly or should be clearly labeled.

## Marketplace Moderation

Used gear listings require extra moderation.

Issues to watch:

- Scams
- Stolen gear
- Fake photos
- Unsafe gear
- Wrong size/category
- Counterfeit products
- Abusive messages

## Reporting System

Users should be able to report:

- Wrong event info
- Fake event
- Cancelled event
- Scam listing
- Offensive content
- Unsafe gear
- Incorrect location

## Admin Dashboard MVP

Minimum admin dashboard features:

- List pending events
- Approve/reject event
- Edit event
- List organizations
- Verify organization
- Remove event
- Feature event
- Review reports

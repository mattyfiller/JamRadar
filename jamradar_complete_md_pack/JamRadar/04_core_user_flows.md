# 04 - Core User Flows

## User Onboarding Flow

1. User downloads app.
2. User creates account or continues as guest.
3. User selects sports:
   - Skiing
   - Snowboarding
   - Skateboarding
   - MTB
   - BMX
   - Indoor training
4. User sets location.
5. User chooses radius:
   - 25 km
   - 50 km
   - 100 km
   - 250 km
   - Custom
6. User chooses event types.
7. User chooses notification style:
   - Instant alerts
   - Daily digest
   - Weekly digest
   - Only saved events
   - Only registration deadlines
8. User lands on personalized event feed.

## Event Discovery Flow

1. User opens Discover tab.
2. App shows nearby events based on preferences.
3. User filters by:
   - Sport
   - Event type
   - Date
   - Distance
   - Cost
   - Skill level
   - Indoor/outdoor
4. User taps event.
5. User views details.
6. User saves event or opens registration link.
7. App reminds user before registration closes or event starts.

## Map Flow

1. User opens Map tab.
2. Events appear as pins.
3. Pins can be filtered by event type.
4. User taps pin.
5. Event preview opens.
6. User saves or views full event.

## Saved Events Flow

1. User saves an event.
2. Event appears in Calendar/Saved tab.
3. User can add it to phone calendar.
4. User gets reminders.
5. User receives updates if event changes.

## Organizer Event Creation Flow

1. Organizer creates profile.
2. Organizer verifies email and organization.
3. Organizer creates an event.
4. Organizer adds:
   - Title
   - Date and time
   - Location
   - Sport
   - Event type
   - Skill level
   - Cost
   - Registration link
   - Description
   - Image/poster
   - Sponsors
   - Categories
5. Organizer publishes event.
6. Event appears in relevant user feeds.
7. Organizer sees analytics.

## Indoor Facility Booking Flow

1. Facility posts available training sessions.
2. User filters for indoor ski/snowboard training.
3. User sees available dates/classes.
4. User books directly or clicks facility booking link.
5. App reminds user before the session.

## Gear Deal Flow

1. User selects gear interests.
2. Shops/brands post deals.
3. User sees relevant gear deals in a dedicated tab or event context.
4. User clicks affiliate link or shop link.
5. App earns affiliate commission or sponsored placement revenue.

## Used Gear Marketplace Flow

1. User lists used gear.
2. User adds photos, size, price, location, condition.
3. App recommends category and fair price range.
4. Buyer filters used gear by sport, size, radius, and condition.
5. Buyer messages seller or purchases through app.
6. App takes a transaction fee if payment is handled in-app.
